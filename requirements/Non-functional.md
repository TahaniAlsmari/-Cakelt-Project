# Non-Functional Requirements

The non-functional requirements define the quality attributes and constraints of the **Cakelt** system, ensuring a high-quality user experience and technical reliability.

| Category | Requirement | Specification |
|:---|:---|:---|
| **Usability** | User Interface | The interface must be intuitive, allowing users to complete a custom order in less than 5 minutes. |
| **Usability** | Accessibility | High contrast and clear typography must be used to ensure readability for all users. |
| **Performance** | Response Time | The system must respond to any user interaction within 2 seconds. |
| **Performance** | Loading Speed | The cake customization preview must render in less than 3 seconds. |
| **Security** | Data Encryption | All personal and payment data must be encrypted using SSL/TLS protocols. |
| **Security** | Authentication | Password hashing must be implemented to protect user account integrity. |
| **Availability** | System Uptime | The application shall be operational and accessible 99.5% of the time. |
| **Scalability** | User Load | The system architecture must handle up to 500 concurrent users during peak seasons. |
